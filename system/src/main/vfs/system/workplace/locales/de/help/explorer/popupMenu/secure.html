Aktiviert eine verschlüsselte Übertragung für die ausgewählte Ressource. Inhalte sollten unbedingt verschlüsselt übertragen werden, wenn sie persönliche Informationen eines Benutzers enthalten, wie dies z.B. bei E-Mail-Formulare der Fall ist.

Nach einem Klick auf den Menüeintrag „Sichern / Exportieren“ öffnet sich ein Dialogfenster in dem die entsprechenden Einstellungen vorgenommen werden können. 

  • Verschlüsselt übertragen via HTTPS
     Wird hier der Wert „Ja“ gesetzt, wird die Datei bei einem Aufruf durch einen Leser verschlüsselt übermittelt, sofern Sie über die Seiten- oder Kopfnavigation aufgerufen wird. Erkennbar ist eine verschlüsselte Seite an dem https:// in der Adresszeile des Browsers. Auch in der Statuszeile des Browsers wird auf die SSL-Verschlüsselung hingewiesen. Der Dialog ist nur dann aktiviert, wenn für die Site, in der sich die Seite befindet, auch ein sicherer Server konfiguriert ist.
  • Intern       
     Wird in dieses Auswahlfeld aktiviert, dann ist die entsprechende Datei nur noch intern, d.h. durch einen OpenCms-Benutzer aufrufbar. Sie können das Dokument nur über den Workplace aufrufen, nicht aber direkt über das Internet. Damit können z.B. die Seitenelemente des Templates vor einem direkten Aufruf geschützt werden:  Wenn diese Funktion am Seitenelement aktiviert wurde, dann kann der Inhalt des Seitenelements nur noch gemeinsam mit dem Template aufgerufen werden.
