Unter „Strukturierten Inhalten“ versteht man im Zusammenhang mit OpenCms das Erstellen von Inhalten unter Verwendung von speziell auf die jeweilige Art konzipierten Eingabeformularen.

Im Kern von OpenCms befinden sich keine Definitionen für Typen von strukturierten Dokumenten (lediglich im templateone - modul). Jedoch kann ein OpenCms System schnell nach Kundenwünschen um strukturierte Dokumenttypen erweitert werden. Spezielle Editoren werden von OpenCms aus den Definitionen der Dokumenttypen erstellt. Sie erlauben eine schnellere Eingabe der Inhalte für solch ein Dokument und versichern, dass die Informationen tatsächlich der geforderten Struktur entspricht.

Was bringen strukturierte Inhalte?

Strutkurierte Inhalte:

  • können flexibler dargestellt werden:
    Wo sollen die News erscheinen? Sollen separat nur die Teaser erscheinen?
  • können schneller Eingegeben werden:
    OpenCms generiert Editoren die für jedes Datenfeld des strukturierten Dokumenttyps ein angepasstes Eingabeelement enthalten.
  • Können aggregiert werden:
    "Gebe mir alle 'news' - Dokumente und stelle Sie hier dar".
  • Sind in einer Form gespeichert, die die Weiterverarbeitung der Informationen auch anderen Applikationen erlaubt (offener Standard: xml - schema).

Der Unterdialog zum Erstellen eines strukturierten Inhaltes enthält bei installierten templateone modul einige vordefinierte Dokumenttypen.

Hinweis: Bitte versuchen Sie nicht, ein Dokument vom Typ "Generischer XML Inhalt" zu erzeugen, da dem System die Beschreibung der Struktur fehlt. Dieses ist vielmehr eine Schnittstelle für Module, die weitere strukturierte Dokumenttypen installieren wollen.
